# Lattice Slots

## Purpose
Defines the six-spoke lattice structure as a documentation anchor.
This file is documentation, not executable code.

## Six Spokes
1. Intake
2. Symbolic
3. Memory
4. Kernel
5. Voice
6. Master

## Meaning
- Intake: preserve raw signal before interpretation
- Symbolic: map shape, tensions, symbols, and structural summary
- Memory: retrieve continuity, recency, and recursive trace
- Kernel: choose path, mode, and stable turn directive
- Voice: realize readable output without uncontrolled reinterpretation
- Master: orchestrate the staged chain and bind packets explicitly

## Status
- The six-spoke / hex structure exists behaviorally in the build
- It is not yet formalized as an explicit geometric runtime object
- This file is the canonical human-readable reference for that structure

## Notes
- Pipeline anchor: intake -> symbolic -> memory -> kernel -> voice
- Master sits as orchestration over the staged chain
- Use this file as the reference point when naming spoke transitions or hex behavior
