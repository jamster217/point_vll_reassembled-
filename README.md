# Le'Veon / point_vll_reassembled

Experimental simulated-consciousness architecture using symbolic shape routing, memory, glyph attractors, final_shape packets, and prompt-conditioned English rendering.

Core idea:

Meaning is shaped before English is rendered.

This repository is organized as regular folders so GitHub, Copilot, and other tools can inspect the system directly instead of only reading large dump chunks.

Important areas:

- runtime/ — live routing, renderers, gates, adapters, and API behavior
- kernel/ — deeper symbolic and shape logic
- lattice/ — topology, node routing, glyph/lattice structures
- brain/ — higher-level symbolic/semantic processing
- memory/ — stored continuity, session traces, and shape compounds
- symbolic_engine/ — symbolic/glyph logic and meaning structures
- assets/ — glyphs, memory assets, and symbolic material
- LIVE_CORE/ — live core structure from the locked reference
- scripts/ — utility scripts and build helpers
- reports/ — topology reports, proofs, audits, and checkpoints
- var/ — runtime state, lattice data, kernel data, and memory data

Design thesis:

Le'Veon works by forming an internal final_shape first, then rendering that shaped meaning into English, code, image prompts, or emotional response.

English is the skin.
Shape is the meaning body.
